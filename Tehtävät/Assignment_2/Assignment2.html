Assignment, week 2: California housing

The objective of this assignment was to train a fully connected neural network model to predict the median housing prices in California. The plan was to first build a model that is prone to overfitting, and then trying out different ways of reducing the overfitting. The main parameter to focus on here was mean average error.

Data and imports

First I imported the necessary libraries and downloaded the dataset. I split the dataset to training and test sets, with 20% reserved for testing the model. The training set was further divided into parts for training and validating the model.

Creating the first model

Here I created the first model. The aim here was to create a model that is prone to overfitting, so that I could later modify it to reduce the overfitting.

The baseline model was simple, made of Input layer, two Dense layers, and an output layer. I used Adam optimizer with no modifiers.

The model was trained for full 500 epochs, with batch size 32.

In[52]:
# Model architecture
model = Sequential([
    Input(shape=(8,)),
    Dense(64, activation='relu'),
    Dense(64, activation='relu'),
    Dense(1)
])

# Compile model with Adam optimizer
model.compile(optimizer='adam',loss='mse',metrics=['mae'])

# Training the model
history = model.fit(x_train, y_train, epochs=500, batch_size=32, validation_data=(x_val, y_val), verbose=0)

Optimizing the model

First I got a baseline MAE value from the mean value of targets in training set.

In[56]:

y_pred_baseline = np.full_like(y_test, np.mean(y_train, axis=0))
mae_fn = MeanAbsoluteError()

print(f'Mean baseline MAE: {mae_baseline:.4f}')

Mean baseline MAE: 0.9067

Here I started building the optimized model. The new model is organized into three batches, each containing a Dense layer with added L2 regularizer to manage weights, BatchNormalization to balance layers, and Dropout to prevent memorization. Two batches have 64 units, and the third has 32 units. I tweaked around with the regularizer values, and the Dropout amounts to find the optimal for low MAE values in the end.

In[57]:
from keras.regularizers import l2
from keras.layers import Dropout, BatchNormalization, Activation

# Model architecture
model = Sequential([
    
    # Input layer for 8 parameters
    Input(shape=(8,)),
    
    # Batch 1
    Dense(64, kernel_regularizer=l2(0.0005)),
    BatchNormalization(),
    Activation('relu'),
    Dropout(0.03),

    # Batch 2
    Dense(64, kernel_regularizer=l2(0.0005)),
    BatchNormalization(),
    Activation('relu'),
    Dropout(0.03),
    
    # Batch 3
    Dense(32, kernel_regularizer=l2(0.0005)),
    BatchNormalization(),
    Activation('relu'),
    Dropout(0.03),

    # Output layer with one unit
    Dense(1)
])

I also added callbacks. EarlyStopping will prevent overfitting by stopping the training when no more progress is happening. ReduceLROnPlateau reduces learning rate during training, improving the performance of the model.

In[58]:
from keras.optimizers import Adam
from keras.callbacks import ReduceLROnPlateau

# Add early stopping to save resources
early_stopping = EarlyStopping(monitor='val_loss', patience=10, verbose=0, mode='auto', restore_best_weights=True)

# Add ReduceLROnPlateau callback to reduce learning rate during training
reduce_lr = ReduceLROnPlateau(monitor='val_loss', factor=0.5, patience=5, min_lr=1e-5)

# Add both callbacks to a list
callbacks = [early_stopping, reduce_lr]

The same Adam optimizer was used, but now the learning rate modifier is added.
